Meeting 2/4: PRD v1

Name: FlightWatch

Problem statement
- Flight ticket prices change frequently and unpredictably
- Users must re-check flights across days or weeks
- Manual tracking is time-consuming and stressful
- Users miss the best price

Goal
- Track selected flights
- Detect lowest point or significant drops
- Notify the user

Success
- User can track at least one route
- System stores historical price data
- User gets a notification when price drops
- App runs reliably for multiple users (push to v2)

Scope
- User account creation optional (v2)
- Show trend first
- If user wants track/save, then account
- Flight tracking (one or more routes)
- Periodic price checks
- Price history storage
- Email notification on drop

Ideas
- Show how many people are looking at this flight

Requirements
4.1 Route Search + Tracking
- Origin airport
- Destination airport
- Departure date (return optional)
- Store tracking request (optional)

4.2 Price Fetching
- Scheduled checks (6h or 12h)
- Save prices with timestamps

4.3 Lowest Price Detection
- Compare to previous lowest
- Compare to user-defined target
- Logic to refind:
  - % drop (example 60%)
  - price range
  - min in time window
  - graph over time

4.4 Notifications
- Email when:
  - price drops below previous minimum
  - price drops by X%

4.5 User account creation (optional)
- Email-based account
- Save previous searches
- Email notifications
